# Multi-Cloud DevSecOps Demo Class Notes

## 1. Introduction

Modern IT companies run hundreds of applications such as websites,
mobile apps, internal tools, and APIs. Large organizations like telecom
companies, banking companies, or large enterprises may run **500--1000
applications**.

To run these applications smoothly, different teams work together:

-   **Development Team** -- Writes application code
-   **QA / Testing Team** -- Tests the application
-   **Security Team** -- Ensures application security
-   **Operations Team** -- Deploys and manages servers

To make application deployment faster, reliable, and automated,
companies use **DevOps and DevSecOps practices**.

------------------------------------------------------------------------

# 2. What is DevOps?

DevOps is a combination of **Development + Operations**.

Simple definition:

> DevOps means running applications in a **cost‑effective, reliable, and
> scalable way**.

Goals of DevOps:

-   Faster software delivery
-   Automation of infrastructure
-   Continuous integration and deployment
-   High availability of applications
-   Reduced manual work

Example:

Instead of manually deploying servers and applications, DevOps engineers
use automation tools to deploy infrastructure and applications
automatically.

------------------------------------------------------------------------

# 3. What is DevSecOps?

DevSecOps means **Development + Security + Operations**.

Simple definition:

> DevSecOps means running applications in a **cost‑effective, reliable,
> scalable, and secure way**.

Security is integrated into every stage of the pipeline.

Key DevSecOps practices:

-   Secure code development
-   Dependency vulnerability scanning
-   Container security
-   Secrets management
-   Infrastructure security

# 9. Infrastructure Design

Two main types of infrastructure design are used.

## Monolithic Infrastructure

Traditional infrastructure architecture where:

-   Application components run on a few large servers.
-   Scaling becomes difficult.

## Landing Zone Architecture

Landing Zone is a **pre‑designed cloud environment** for hosting
applications.

It includes:

-   Identity & Access Management
-   Security policies
-   Networking
-   Governance
-   Compliance
-   Monitoring
-   Backup & Disaster Recovery

Two common types:

-   Normal Landing Zone
-   Hub-Spoke Landing Zone

Landing zones follow **Azure Cloud Adoption Framework**.
