# 🚀 Class 3 – Computer Evolution & Operating Systems

---

## 🧠 1. Basic Components of a Computer

- **CPU (Central Processing Unit)** → Brain of the computer  
- **Memory (RAM)** → Temporary storage  
- **Input Devices** → Keyboard, Mouse  
- **Output Devices** → Monitor, Printer  

👉 Example: **ENIAC (1940s)** – First electronic computer  

---

## ❗ 2. Initial Problem

How to tell the computer **what task to perform?**

---

## 🔌 3. Direct Wiring Programming

- Programs were set using **wires and switches**
- Every new task required **physical rewiring**
- Very complex and time-consuming  

👉 Meaning:  
> Programming was done at hardware level (no software)

---

## 💡 4. Punch Card Programming

- Programs stored as **holes in cards**
- Insert card → Run program  

### Advantages:
- Reusable  

### Disadvantages:
- Slow  
- Error-prone  

---

## 💻 5. Machine Language (Binary)

- Computer understands only:
  - **1 (ON)**
  - **0 (OFF)**  

### Problem:
- Very difficult for humans to write  

---

## ⚙️ 6. Assembly Language

- Introduced human-readable commands:

MOV → Move data  
ADD → Addition  
SUB → Subtraction  

### Key Points:
- Easier than binary  
- Converted to machine code using **Assembler**

---

## 🏢 7. Early Systems

### IBM (1950s–60s)
- Developed early operating systems:
  - GM-NAA I/O  
  - OS/360  

### Bell Labs (1969)
- Developed **UNIX**
- Base for modern operating systems  

---

## 💰 8. Why Early Systems Were Expensive

- License: $1000 – $10000+  
- Expensive hardware (mainframes)  
- High maintenance cost  

### Used by:
- Research labs  
- Big companies  
- Universities  

---

## 💻 9. Microsoft Era (1980s)

### MS-DOS
- Cost: ~$40–$100  
- Bundled with PCs  

👉 Made computers affordable for individuals  

---

### Windows (95, 98)

- Introduced **GUI (Graphical User Interface)**
- Mouse-based interaction  
- Easy to use  

---

## 🐧 10. Linux (1991 – Linus Torvalds)

- Inspired by UNIX (not a copy)  
- Introduced **Linux Kernel**  

---

## 🔓 Open Source vs Closed Source

### Closed Source
- Code not visible  
- Example: Windows  

### Open Source
- Code publicly available (GitHub)  
- Community-driven development  

### Linux Features:
- Free  
- Secure  
- High performance  

---

## ❤️ 11. OS vs Kernel

### Operating System (OS)
Includes:
- GUI  
- Applications  
- File System  
- Kernel  

### Kernel
- Core part of OS  
- Direct interaction with hardware  

### Flow:
Application → OS → Kernel → Hardware  

---

## 🔌 12. Kernel and Device Drivers

- Kernel communicates with hardware using **drivers**

### Examples:
- Printer → Printer Driver  
- Monitor → Display Driver  

---

## 🛠️ 13. Open Source Tools (Linux Ecosystem)

- Linux  
- Kubernetes  
- Terraform  

👉 Widely used in DevOps  

---

## 🐧 14. Linux-Based Operating Systems

- Ubuntu  
- Kali Linux  
- Parrot OS  
- Android  

---

## 🖥️ 15. GUI vs CLI

### GUI (Graphical User Interface)
- Easy to use (click-based)  
- Heavy  

### CLI (Command Line Interface)
- Command-based  
- Lightweight and fast  

👉 Used in DevOps & Cloud environments  

---

## 🎯 Final Summary

- Early programming was hardware-based  
- Evolution: Binary → Assembly → Operating Systems  
- UNIX formed the base of modern OS  
- Microsoft made computers accessible  
- Linux introduced open-source revolution  
- Kernel is the core of OS  
- CLI is essential for DevOps
