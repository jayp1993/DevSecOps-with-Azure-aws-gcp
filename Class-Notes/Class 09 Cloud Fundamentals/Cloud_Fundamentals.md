# ☁️ 1. Deployment Models of Cloud Computing

## 🔹 Public Cloud

Public cloud is a model where multiple customers share the same infrastructure provided by a cloud vendor.

**Examples:** AWS, Microsoft Azure, Google Cloud

**Explanation:**
Infrastructure is physically shared but logically isolated, ensuring data security.

**Key Characteristics:**

* Multi-tenant environment
* Owned by cloud provider
* Access via internet
* Highly scalable
* Cost-effective (Pay-as-you-go)

**Analogy:**
Like staying in a hotel – shared building, private rooms

---

## 🔹 Private Cloud

Private cloud is dedicated to a single organization.

**Explanation:**
Infrastructure is not shared, offering higher control and security.

**Key Characteristics:**

* Single-tenant
* Fully dedicated infrastructure
* High security
* High cost
* Limited scalability

**Analogy:**
Like owning a private house

---

## 🔹 Hybrid Cloud

Combination of public and private cloud.

**Explanation:**
Sensitive data stays in private cloud, scalable workloads run on public cloud.

**Key Characteristics:**

* Mix of both environments
* Flexible
* Balanced cost & security

**Analogy:**
Using both home and hotel as needed

---

# 📊 Deployment Model Comparison

| Feature        | Public Cloud | Private Cloud      | Hybrid Cloud |
| -------------- | ------------ | ------------------ | ------------ |
| Tenancy        | Multi-tenant | Single-tenant      | Mixed        |
| Infrastructure | Shared       | Dedicated          | Combination  |
| Ownership      | Provider     | Organization       | Both         |
| Hardware       | Shared       | Dedicated          | Partial      |
| Isolation      | Logical      | Physical + Logical | Controlled   |
| Cost           | Low          | High               | Medium       |
| Access         | Internet     | Private Network    | Both         |
| Scalability    | Very High    | Limited            | Flexible     |

# ⚙️ 2. Types of Cloud Services

## 🔹 IaaS (Infrastructure as a Service)

**Provides:**

* Virtual Machines
* Storage
* Networking

**You manage:**

* OS
* Middleware
* Applications

**Examples:** Azure VM, AWS EC2

**Analogy:**
Empty house → you set everything

---

## 🔹 PaaS (Platform as a Service)

**Provides:**

* Platform (OS + Runtime)

**You manage:**

* Application code

**Examples:** Azure App Service

**Analogy:**
Ready kitchen → just cook

---

## 🔹 SaaS (Software as a Service)

**Provides:**

* Complete software

**You manage:**

* Usage only

**Examples:** Microsoft 365, Gmail

**Analogy:**
Restaurant → just eat

---

# 🧱 3. Responsibility Model

## 🔹 On-Premises

You manage everything:

* Hardware
* Network
* OS
* Middleware
* Application

---

## 🔹 IaaS

* Provider → Hardware, Network
* You → OS, Middleware, Application

---

## 🔹 PaaS

* Provider → Infrastructure, OS, Runtime
* You → Application

---

## 🔹 SaaS

* Provider manages everything
